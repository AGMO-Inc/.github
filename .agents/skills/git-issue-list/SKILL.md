# GitHub Issue 목록 조회

이 스킬은 현재 연결된 레포의 이슈를 조회하고, 간단 설명을 포함한 표를 출력한다.

## 1) 조회

1. `gh`가 설치/인증되어 있는지 확인한다.
2. 기본 조회 범위는 `open` 이슈다.
3. 사용자가 요청하면 `closed` 또는 `all`로 변경한다.

예시:

```bash
gh issue list --state open --limit 100 --json number,state,title,url,body
```

## 2) 간단 설명 추출

1. `body`의 첫 문장(또는 첫 줄)을 간단 설명으로 사용한다.
2. 본문이 없으면 `-`로 표시한다.
3. 너무 길면 80자 내외로 잘라서 표시한다.

## 3) 출력 형식 (필수)

반드시 아래 표 형식으로 출력한다.

```text
| # | 상태 | 제목 | 간단 설명 | 링크 |
|---:|---|---|---|---|
| 12 | OPEN | [Feature] 예시 제목 | 공용 스킬 구조를 정리한다. | https://github.com/org/repo/issues/12 |
| 11 | OPEN | [Bug] 예시 제목 | 로그인 실패 재현 경로가 있다. | https://github.com/org/repo/issues/11 |
```

## 4) 안전 규칙

1. 이 스킬은 조회 전용이다. 이슈 생성/수정/삭제를 수행하지 않는다.
2. 이슈가 0건이면 표 헤더만 출력하고 "조회 결과 없음"을 함께 안내한다.
