# Issue Start Workflow

이 스킬로 이슈 기반 작업 시작을 표준화한다.

## 1. 이슈 조회

1. 사용자에게 이슈 번호를 확인한다.
2. gh 명령어로 현재 레포의 이슈를 조회한다. (gh 가 없다면 사용자에게 다운로드 및 인증을 요청한다.)
3. 제목, 본문, 라벨(유형), 체크리스트 항목을 추출한다.

## 2. TODO-Issue.md 갱신

1. `TODO-Issue.md`를 현재 이슈 기준으로 덮어써 최신화한다. 해당 파일이 없다면 새로 생성한다.
2. 최소 항목을 포함한다.
   - 레포
   - 이슈 번호
   - 제목
   - 링크
   - 실행 과제 체크리스트
3. 실행 과제는 이슈 본문과 수용 기준에서 직접 파생한다.

## 3. 브랜치 생성

1. 이슈 유형을 라벨/제목에서 판별한다.
2. 규칙에 맞는 브랜치를 생성한다.
   - Bug: `fix/영향도메인-#이슈번호`
   - Feature: `feature/영향도메인-#이슈번호`
   - Task: `feature/영향도메인-#이슈번호`
3. 영향 도메인이 불명확하면 사용자에게 짧게 확인 후 생성한다.

## 4. 결과 보고

1. 갱신된 `TODO-Issue.md` 핵심 항목을 요약한다.
2. 생성된 브랜치명을 공유한다.
3. 다음 단계(구현 시작 또는 커밋 준비)를 제안한다.
